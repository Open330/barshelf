# R10 — 위젯 노드 접근성 라벨 (accessibilityLabel)

## 목표
위젯 제작자가 의미 있는 심볼/버튼/행에 명시적 VoiceOver 라벨을 줄 수 있는
옵셔널 노드 필드 `accessibilityLabel` 추가. 없을 때는 기존 App 하드닝 동작
(데코레이티브 심볼 숨김, 파일 썸네일 파일명 라벨) 보존.

## 변경 파일
- `Sources/MenubucketCore/UINode.swift`
  - `UINode`에 `public var accessibilityLabel: String?` 추가 (`drag` 뒤).
  - `public init(...)` **맨 끝**에 `accessibilityLabel: String? = nil` 추가 —
    기존 호출부(App/Adapter/Scaffold) 시그니처 호환 유지.
  - Equatable(합성)/Sendable(기존 extension) 그대로 유지.
- `Sources/MenubucketApp/Renderer/ViewTreeRenderer.swift`
  - `NodeView.body`를 `decorated`(기존 drag/layout 로직) + 조건부
    `.accessibilityLabel(label)`로 분리. 라벨 nil이면 기존 동작 완전 보존.
- `schema/uinode-0.1.json`
  - `commonNode.properties`에 `accessibilityLabel` (옵셔널 string) 추가.
    기존 nested `accessibility` 객체와는 별개(플랫 필드).
- `docs/WIDGET-SPEC.md`
  - 공통 필드 예제에 `accessibilityLabel` 추가 + "### 접근성" 섹션(버튼/이미지 예시).
- `Tests/MenubucketCoreTests/UINodeTests.swift`
  - +4 테스트: 디코딩, 기본 nil, 라운드트립, 알 수 없는 필드와 공존.

## 검증
- `jq empty schema/uinode-0.1.json` → OK
- `swift build` → Build complete
- `swift test` → 171 passed (이전 167 → +4), 0 실패

## 설계 노트
- 스키마에 이미 `accessibility:{label,hint,value}` nested 객체가 있으나, 태스크
  요구대로 플랫 `accessibilityLabel` 노드 필드를 UINode(Swift)와 렌더러에 매핑.
  두 가지가 공존(nested는 계약상 예약, 플랫은 실제 렌더 경로).
- 라벨은 Group 대신 `decorated` 하위 뷰에 적용해 컨테이너 노드에서도 자연스럽게
  단일 접근성 요소로 합쳐지도록 함.
